## Objectif du TP

Ce travail pratique avait pour objectif de :
- Comprendre les principes SOLID de la programmation orientée objet
- Identifier du code non conforme à ces principes
- Appliquer un refactoring pour rendre le code plus maintenable, extensible et lisible
- Mettre en pratique ces principes à travers des exemples concrets en Java.

## Principes SOLID étudiés

Les cinq principes SOLID sont traités dans ce projet :
.
1. S – Single Responsibility Principle (SRP)  
2. O – Open/Closed Principle (OCP)  
3. L – Liskov Substitution Principle (LSP)  
4. I – Interface Segregation Principle (ISP)  
5. D – Dependency Inversion Principle (DIP)  

Chaque principe dispose :
- d’exemples de code non conformes,de leur refactoring et enfin d’un README explicatif dans les dossiers correspondants



## Contenu des dossiers SOLID

Chaque dossier du principe contient :
- plusieurs fichiers Java correspondant aux exemples vus en salle
- une version avant refactoring
- une version après refactoring
- un fichier README.md expliquant :
  - le principe
  - le problème du code initial
  - la solution appliquée
